Project 03

Austin Eats — Building the Data Pipeline

↗ Live Dashboard

The Story

01

Started with Yelp API — hit a business email wall. Pivoted to Google Places (New), which turned out to have better data anyway.

02

Pulled 230 restaurants across 12 cuisine types using Python. First clean showed 49 restaurants dumped into "Other Austin" — a data quality problem worth fixing.

03

Rebuilt the neighborhood mapping using a full Austin zip code dictionary. Reduced "Other Austin" to zero across 20 distinct neighborhoods.

04

Designed the hidden gem scoring algorithm to surface underrated spots over famous ones. Published to Tableau Public with live filters.

Data limitation acknowledged: 58 of 230 restaurants had no price data from Google Places. Labeled "Unknown" — honest data beats clean-looking data.

Stack & Output

Google Places API (New) — 230 restaurants
Python (requests, pandas, dotenv)
20 Austin neighborhoods mapped by zip code
Tableau Public — live interactive dashboard

Hidden Gem Scoring Algorithm

(rating × 20) −
(min(review_count, 1000) ÷ 50)

High rating + low review count = underrated gem. Caps review count at 1,000 so mega-popular spots aren't unfairly tanked.

Approach

How I work
through a problem.

Every project follows the same discipline: start with a real question, find the right data, and don't stop until the answer is actionable.

01

Define the Marketing Question

Before touching data, I identify what decision the analysis needs to support — budget reallocation, channel expansion, or measurement gap identification.

02

Source & Validate Data

Pull from APIs, public datasets, or earnings reports. Clean in Python or Excel. Document every assumption and flag estimated vs. verified figures.

03

Analyze Against Benchmarks

Performance only means something in context. I compare channel metrics against industry benchmarks — CTR, ROAS, CAC — to identify what's over- or under-performing.

04

Deliver Actionable Recommendations

Every output ties back to a decision. I present findings as recommendations with expected outcomes — not just observations.
